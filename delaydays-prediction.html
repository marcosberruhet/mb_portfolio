Predicción de Días de Demora en Pagos

Modelo de Machine Learning para estimar el desvío de pago en días sobre facturas emitidas en contextos B2B.

🧾 Sobre el Proyecto

Este proyecto busca predecir el desvío en días entre la fecha de vencimiento y la fecha real de pago de una factura (delay_days). Utiliza datos históricos de operaciones entre empresas para asistir decisiones financieras.

El dataset incluye fechas clave (emisión, vencimiento, pago), montos, tipo de documento, y datos de comprador/emisor. Se realizó una limpieza y selección rigurosa de variables para asegurar la calidad del modelo.

⚙️ Detalles Técnicos

La función prepare_data() automatiza la limpieza del dataset. Las tareas incluyeron:

  • Conversión de fechas a formato datetime.
  • Extracción de variables como mes y día de semana.
  • Eliminación de columnas irrelevantes o de alta cardinalidad.
  • One-hot encoding para categorías.
  • Imputación de nulos y escalado de variables numéricas.
  • Remoción de columnas constantes.

El modelo es una red neuronal densa entrenada con TensorFlow/Keras. Se dividió el dataset 80/20 y se optimizó con MAE (Mean Absolute Error), obteniendo un error medio de aproximadamente 3.4 días.

📊 Resultados y Evaluación

Desempeño del modelo en entrenamiento y validación:

Curva de evolución del MAE: El error convergió a ~3.4 días, mostrando buena generalización y sin sobreajuste evidente.

Histograma de Errores

Errores de predicción: Más del 80 % de las predicciones están dentro del rango de ±5 días. Algunos outliers superan los 20 días, lo cual podría refinarse con más ingeniería de variables.

El modelo aporta valor en la evaluación de riesgo y planificación financiera al anticipar el comportamiento de pago.
